Try out `starlight-plugin-icons` with these examples.

## Sidebar with icons

Add an `icon` property to sidebar items using an icon class.

Format: `i-<collection>:<name>`

```js title="astro.config.mjs" "icon: 'i-ph:rocket-launch-duotone'"
import Icons from 'starlight-plugin-icons'

export default {
  integrations: [
    Icons({
      sidebar: true,
      starlight: {
        sidebar: [
          {
            label: 'Guide',
            items: [
              { icon: 'i-ph:rocket-launch-duotone', label: 'Intro', slug: 'guide/intro' },
            ],
          },
        ],
      },
    }),
  ],
}
```

Result:

<Image src={sidebarDemo} alt="Sidebar with icon" />

<br />

<Aside type="note">
  - Make sure to install the Iconify collection that you use: <Link href="https://icones.js.org/">Icon Gallery</Link>.
  - Want to tweak spacing or sizes? See the CSS variables in [Configuration](./guides/configuration).
</Aside>

## Code blocks with icons

Once the plugin is installed, code fences automatically get a small icon.
Icons are chosen either by the language or by the filename in the block's `title`.

<Steps>

1. ##### Enable the features

    ```js title="astro.config.mjs" ins={2-3}
    Icons({
      codeblock: true,
      extractSafelist: true,
      starlight: { ... },
    })
    ```

2. ##### Add the cache to `.gitignore`

    ```txt title=".gitignore"
    .material-icons-cache/*
    !.material-icons-cache/material-icons-safelist.json
    ```

</Steps>

### Based on language

Example source

````mdx
```ts title="app.ts"
console.log('hi')
```
````

Rendered

```ts title="app.ts"
console.log('hi')
```

<br />

<Aside type="tip" icon="i-ph:lightbulb-bold">
  This uses a language-to-icon mapping from <Link href="https://github.com/material-extensions/vscode-material-icon-theme">VS Code Material Icon theme</Link>.
</Aside>

### Based on filename

````mdx
```md title="README.md"
# README
```
````

Rendered

```md title="README.md"
# README
```

<br />

<Aside type="tip" icon="i-ph:lightbulb-bold">
  - To view the full list, see the `.material-icons-cache\material-icons.json` file.
  - If no specific match is found, it falls back to the language-based icon.
</Aside>
